# Memo

Minimal blog/card app built with Next.js 16, Bun, Tailwind CSS v4, Supabase, and Azure Blob Storage.

## Stack

- Next.js 16 App Router
- Bun
- TypeScript + ESLint
- Tailwind CSS v4
- Supabase Postgres for blog posts
- Azure Blob Storage for post images via short-lived SAS URLs

## App Structure

- `app/page.tsx`: home page, server-rendered blog card list
- `app/components/add-post-form.tsx`: add blog button and modal form
- `app/actions.ts`: server actions for Supabase mutations
- `app/api/upload-url/route.ts`: backend endpoint that creates upload SAS URLs
- `app/lib/supabase.ts`: Supabase client and Azure image URL helper
- `scripts/setup-cors.ts`: helper script for Azure Blob Storage CORS setup

## Local Development

Install dependencies and start the app:

```bash
bun install
bun run dev
```

Open [http://localhost:3000](http://localhost:3000).

## Required Environment Variables

Create a local `/.env.local` file inside `memo/` with these variables:

```bash
SUPABASE_URL=
SUPABASE_ANON_KEY=
AZURE_STORAGE_ACCOUNT_NAME=
AZURE_STORAGE_ACCOUNT_KEY=
AZURE_STORAGE_CONTAINER_NAME=
AZURE_BLOB_CONTAINER_URL=
```

## Security Notes

- Do not commit `.env`, `.env.local`, or any other secret-bearing env files.
- `AZURE_STORAGE_ACCOUNT_KEY` must stay server-side only.
- The Azure container is intended to stay private.
- The browser should only receive short-lived SAS URLs generated by `POST /api/upload-url`.
- If any secrets were previously committed, rotate them before pushing.

## Safe To Push

This repo is safe to push only if:

- secret env files are not tracked by Git
- storage keys and Supabase keys are not hardcoded in source files
- local-only files remain ignored by `.gitignore`

Before pushing, verify:

```bash
git status --ignored
```

and confirm that `.env.local` is ignored and not staged.
